Why This Architecture

Rather than spinning up a paid cloud server, this application runs on an Android device running Termux — a full Linux environment on a phone. This was a deliberate decision rooted in understanding how large language models like Claude actually work.

Claude operates as a stateless inference engine — each request is self-contained, context is passed in the prompt, and no persistent state lives inside the model itself. The same philosophy applies here: the backend is stateless (Express + JWT), all state lives in MariaDB, and the frontend is a pure static build. This means the entire stack can run on any compute — including a phone.

Cloudflare Tunnel exposes the local server without port forwarding or a static IP — the same pattern used in edge deployment scenarios where compute is close to data but not publicly addressable.

This GitHub Pages URL acts as a stable entry point — a thin indirection layer decoupling the permanent submission URL from the dynamic tunnel beneath it. When the tunnel restarts, this page updates automatically via the GitHub API. This mirrors DNS and service discovery in production: stable names pointing to dynamic infrastructure.
